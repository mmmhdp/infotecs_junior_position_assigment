# Задание 1

## Описание

Есть микросхема EEPROM-памяти с интерфейсом SPI (например, **25LC040A**).

Имеется драйвер, позволяющий управлять линиями микросхемы через переключение битов (*bit-banging*).

Необходимо реализовать **C++ класс (или набор классов)** для работы с микросхемой посредством данного драйвера.

## Функциональные требования

Требуется реализовать:

- Запись данных по адресу:
  - одного бита
  - одного байта
  - массива байт
- Чтение данных по адресу:
  - одного бита
  - одного байта
  - массива байт

Также необходимо описать, **что изменится**, если потребуется реализовать драйвер для работы с **NOR-памятью**
(например, **W25Q128**).

## Документация

- Все методы, константы, данные, перечисления и т.п. должны быть **документированы**
- Предпочтительный формат документации — **Doxygen**

## Требования к коду

- Декларации классов должны быть **синтаксически корректными**
- Код должен **компилироваться** (реализация может быть частичной, но корректной)

# Ответ на вопрос о смене девайса для работы с памятью

Фактически, при смене девайса на NOR-память (например, W25Q128) 
требуется **новая реализация интерфейсов памяти** (`i_memory_device_api`) и,
при необходимости, SPI (`i_chip_spi_api`).

Если углубляться в детали, то для NOR-памяти нужно учитывать:

- 24-битные адреса вместо 9–16 бит;
- программирование страниц (page write), так как запись байта напрямую невозможна;
- необходимость стирания секторов перед записью;
- более медленные операции записи и чтения статуса;
- потенциально более сложную работу с битами (read-modify-write по странице).

Таким образом, структура классов остаётся прежней, 
но реализация всех методов должна соответствовать ограничениям NOR-памяти.
